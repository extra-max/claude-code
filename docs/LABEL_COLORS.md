# Label Organization Guide

## Label Categories

| Label Name | Category | Description |
|------------|----------|-------------|
| `bug` | Issue Type | Something isn't working |
| `enhancement` | Issue Type | New feature or request |
| `documentation` | Issue Type | Improvements or additions to documentation |
| `question` | Issue Type | Further information is requested |
| `duplicate` | Issue Type | This issue or pull request already exists |
| `platform:macos` | Platform | Issues specific to macOS |
| `platform:linux` | Platform | Issues specific to Linux |
| `platform:windows` | Platform | Issues specific to Windows |
| `area:core` | Area | Core functionality |
| `area:tools` | Area | Tooling and utilities |
| `area:ide` | Area | IDE integration |
| `area:tui` | Area | Terminal User Interface |
| `area:model` | Area | Model behavior and output |
| `area:security` | Area | Security related issues |
| `area:mcp` | Area | Model Context Protocol |
| `area:auth` | Area | Authentication and authorization |
| `area:packaging` | Area | Packaging and installation |
| `area:api` | Area | API interactions |
| `has repro` | Status | Reproduction steps are available |
| `memory` | Status | Memory usage related |
| `perf:memory` | Status | Memory performance improvements |
| `external` | Status | Issues related to external dependencies |

## Usage Guidelines

- **Issue Type**: Every issue should have at least one label from this category (bug, enhancement, etc.).
- **Platform**: Apply if the issue is specific to a certain OS.
- **Area**: Use to denote which part of the codebase is affected.
- **Status**: Use to track the state or specific attributes of the issue.
